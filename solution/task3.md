# Задание 3

## Цель архитектуры
Спроектировать изолированную корпоративную сеть с одним сервером приложений, реляционной БД и S3-хранилищем. Подходит для CRM/порталов с фокусом на безопасность и масштабирование.
## Инфраструктурная связка продуктов

| Компонент | Продукт Selectel | Роль |
|-----------|------------------|------|
| **Вычисления** | [Облачные серверы](https://docs.selectel.ru/cloud/servers/) | Сервер приложений + bastion/VPN |
| **Сеть** | [Приватные сети (VPC)](https://docs.selectel.ru/cloud/networks/) + [Глобальный роутер](https://docs.selectel.ru/network/global-router/) | L3-изоляция ресурсов |
| **БД** | [Облачные базы (Managed PostgreSQL/MySQL)](https://docs.selectel.ru/cloud/databases/) | Структурированные данные |
| **Хранилище** | [Объектное хранилище S3](https://docs.selectel.ru/storage/s3/) | Файлы, бэкапы по S3-API |

## Логическая схема
На схеме показано движение трафика: внешние запросы фильтруются на Firewall, а прямое управление ресурсами доступно только через Bastion-хост. База данных полностью изолирована и не имеет публичного интерфейса.
```
ИНТЕРНЕТ 
   │
[Облачный Firewall]
   │
[Bastion/VPN] ────── (SSH/API для админов)
   │
   ▼
[Сервер приложения] ◄──────┐ 
   │       │               │ (Приватная VPC L3)
   │       │               │ 
   │       ▼               ▼
   │  [Managed PostgreSQL] [S3-бакет]
   │  (Только private IP)  (S3-API Endpoint)
   └───────────────────────┘
```

## Преимущества

**Сетевая изоляция базы данных**: База данных доступна только по приватным IP-адресам внутри VPC. Публичные адреса для managed PostgreSQL отсутствуют — злоумышленник не может подключиться напрямую из интернета.

**Безопасное администрирование**: Управление инфраструктурой происходит строго через Bastion-хост по SSH-туннелю. Административные порты сервера приложений закрыты для внешнего мира, что сводит поверхность атаки к одной защищенной точке входа

**Сервер не забивается файлами**: Файлы, изображения и бэкапы хранятся в S3. Локальный диск сервера содержит только ОС и приложение (менее 20 ГБ). Это обеспечивает быстрое клонирование VM при масштабировании нагрузки — новая инстанция готова за 2 минуты.

**Масштабирование**: Managed-сервисы позволяют увеличивать CPU/RAM базы данных или объем S3 одним кликом в панели my.selectel.ru без остановки сервисов. Сеть VPC остается неизменной при любом росте инфраструктуры.
